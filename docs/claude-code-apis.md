# Claude Code APIs

Runtime data sources used by this project.

---

## 1. Statusline stdin JSON

Claude Code passes a JSON object to `statusline.sh` on every status bar tick (via stdin). The shell script reads it once at startup and exports the relevant fields as env vars for element scripts.

### Schema

```json
{
  "model": {
    "display_name": "claude-sonnet-4-6"
  },
  "context_window": {
    "used_percentage": 12.4,
    "context_window_size": 200000
  },
  "cost": {
    "total_cost_usd": 0.0342,
    "total_duration_ms": 18450
  },
  "workspace": {
    "current_dir": "/Users/alice/project",
    "project_dir": "/Users/alice/project"
  },
  "session_id": "abc123",
  "transcript_path": "/Users/alice/.claude/projects/-Users-alice-project/abc123.jsonl"
}
```

### Field notes

| Field | Notes |
|---|---|
| `model.display_name` | Human-readable model name, e.g. `claude-sonnet-4-6` |
| `context_window.used_percentage` | Nullable â€” `null` before the first API call in a session |
| `context_window.context_window_size` | `200000` normally; `1000000` when extended context is active |
| `cost.total_cost_usd` | Cumulative USD cost for the session |
| `cost.total_duration_ms` | Cumulative wall-clock time of all API calls in the session |
| `workspace.current_dir` | CWD of the Claude Code process |
| `workspace.project_dir` | Root of the detected project |
| `session_id` | Unique identifier for the current session |
| `transcript_path` | Absolute path to the JSONL transcript for the session |

### Env var exports

`statusline.sh` parses stdin once and exports these vars so element scripts can read them without re-parsing:

| Env var | Source field |
|---|---|
| `CLAUDE_MODEL` | `model.display_name` |
| `CLAUDE_CONTEXT_PCT` | `context_window.used_percentage` |
| `CLAUDE_CONTEXT_SIZE` | `context_window.context_window_size` |
| `CLAUDE_COST_USD` | `cost.total_cost_usd` |

## 2. Hook stdin JSON

Claude Code passes a JSON object to hook scripts on each event. The hook script reads it from stdin.

### Events and fields

**All events** include:

```json
{
  "hook_event_name": "PostToolUse",
  "session_id": "abc123",
  "transcript_path": "/Users/alice/.claude/projects/-Users-alice-project/abc123.jsonl",
  "cwd": "/Users/alice/project"
}
```

**`PostToolUse`** additionally includes:

```json
{
  "hook_event_name": "PostToolUse",
  "tool_name": "Bash",
  "tool_response": {
    "is_error": false
  }
}
```

**`Notification`** additionally includes:

```json
{
  "hook_event_name": "Notification",
  "message": "Claude needs your permission to run a command.",
  "title": "Permission Required",
  "notification_type": "permission_prompt"
}
```

### `notification_type` values

| Value | Trigger |
|---|---|
| `permission_prompt` | Claude is asking to run a tool that requires approval |
| `idle_prompt` | Claude is idle and waiting for user input |
| `auth_success` | Authentication completed successfully |
| `elicitation_dialog` | Claude is asking the user a question via a dialog |

### Registered hooks in this project

| Event | Script | Purpose |
|---|---|---|
| `PostToolUse` | `hooks/sounds.sh` | Play error sound when `tool_response.is_error` is true |
| `Stop` | `hooks/sounds.sh` | Play stop sound when Claude finishes a turn |
| `Notification` | `hooks/sounds.sh` | Play notification sound on any notification event |

## 3. Session JSONL format

Each session writes a `.jsonl` file at `~/.claude/projects/**/*.jsonl`. Every line is a self-contained JSON object. Element scripts that need session-level metrics (model, token burn rate, session cost, streak) read these files directly via inline Python.

### Message object

```json
{
  "type": "assistant",
  "timestamp": "2026-03-15T14:23:01.456Z",
  "message": {
    "model": "claude-sonnet-4-6",
    "usage": {
      "input_tokens": 4210,
      "output_tokens": 312,
      "cache_creation_input_tokens": 0,
      "cache_read_input_tokens": 3800
    }
  }
}
```

### Field notes

| Field | Notes |
|---|---|
| `type` | `"user"` or `"assistant"` |
| `timestamp` | ISO 8601 with milliseconds |
| `message.model` | Only present on `assistant` entries |
| `message.usage.input_tokens` | Tokens sent to the model (includes cache hits) |
| `message.usage.output_tokens` | Tokens generated by the model |
| `message.usage.cache_creation_input_tokens` | Tokens written to the prompt cache |
| `message.usage.cache_read_input_tokens` | Tokens served from the prompt cache |

### Usage in this project

Elements that read JSONL directly:

| Element | What it reads |
|---|---|
| `model.sh` | Latest `message.model` from the most recent assistant entry |
| `context-pct.sh` | Latest `usage` to compute context fill percentage |
| `burn-rate.sh` | `output_tokens` over a time window to compute tokens/min |
| `session-cost.sh` | Cumulative token counts Ă— per-token pricing |
| `streak.sh` | Count of consecutive assistant turns without a user interrupt |
